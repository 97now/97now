| 열(Column) | 의미 | 이걸 보면 알 수 있는 것 |
| --- | --- | --- |
| **Field** | 테이블의 필드명 | SBOOK 안의 각 필드 이름 (`CARRID`, `CONNID`, `CUSTOMID` 등) |
| **Data element** | 데이터 요소 | 필드의 의미·도메인·검색 도움말을 정의한 DDIC 객체 |
| **Foreign Key**  | 체크박스 (있으면 외래키 연결됨) | 표시되어 있으면 다른 테이블의 필드를 참조(= 관계가 있음) |
| **Check table** | 참조 대상 테이블 이름 | 이 필드가 어떤 테이블의 어떤 필드를 참조하는지 |
| **Origin of Input Help** | 입력 도움말 생성 방식 | F4 도움말이 어디서 오는지 (고정값, 외래키, Search Help 등) |

| **Domain** | 값의 형식, 길이, 허용 범위 | 도메인 레벨에서 데이터 타입/길이/고정값 제어 |


## “Origin of Input Help” 열 — 뜻 정리표

| 문구 | 의미 | 실제 동작 방식 (F4 눌렀을 때) |
| --- | --- | --- |
| **Input help implemented with check table** | **체크 테이블(Check Table)** 을 참조해서 값 목록을 보여줌 | 예: `CARRID` 필드에서 F4 → `SCARR` 테이블의 항공사 목록이 뜸 |
| **Input help implemented with fixed values** | **도메인(Domain)** 안에 미리 정의된 **고정값(Fixed Values)** 을 보여줌 | 예: `CLASS` 필드 → 도메인에 정의된 `F`, `C`, `Y` 값이 뜸 |
| **Input help based on data type** | **데이터 타입(Domain Type)** 자체에 내장된 도움말을 사용 | 예: 날짜형(DATS)은 캘린더 팝업이 뜨고, 숫자는 숫자 입력창이 뜸 |
| **Explicit search help interface to field** | 필드에 직접 연결된 **명시적 검색 도움말(Search Help)** 사용 | 예: `BOOKID` 필드 → `SBOOK` Search Help가 직접 연결되어 있음 |
| **Input help implemented with search help** | **데이터 요소(Data Element)** 수준에서 연결된 **Search Help** 사용 | 예: 데이터 요소 `S_CUSTOMER` → `SCUSTOM` Search Help 사용 |

### 1 “Input help implemented with check table”

**뜻:**

> 이 필드는 Foreign Key(외래키) 로 다른 테이블을 참조하므로,
> 
> 
> F4 도움말을 누르면 **그 참조 테이블의 값 목록**이 자동으로 뜬다.
> 

**예시:**

| 필드 | 체크 테이블 | 결과 |
| --- | --- | --- |
| `CARRID` | SCARR | 항공사 목록 표시 |
| `CONNID` | SPFLI | 항공편 번호 목록 표시 |
| `CUSTOMID` | SCUSTOM | 고객 ID 목록 표시 |

즉, “데이터를 참조하고 있으니, 참조 테이블의 값을 보여줄게” 라는 뜻.

---

### 2 “Input help implemented with fixed values”

**뜻:**

> 이 필드는 도메인(Domain) 안에 고정된 값 목록(Fixed values) 을 갖고 있어서,
> 
> 
> F4를 누르면 그 값들이 나열됨.
> 

**예시:**

| 필드 | 도메인 | 고정값 | 설명 |
| --- | --- | --- | --- |
| `CLASS` | S_CLASS | F / C / Y | First / Business / Economy |
| `SMOKER` | S_SMOKER | X / (공백) | 흡연자 / 비흡연자 |
| `INVOICE` | S_FLAG | X / (공백) | 청구 여부 |
| `CANCELLED` | S_CANCEL | X / (공백) | 취소 여부 |

즉, 이건 “테이블 참조 없이 고정된 코드값”을 직접 보여주는 경우.

---

### 3 “Input help based on data type”

**뜻:**

> 이 필드에는 별도의 참조나 고정값이 없지만,
> 
> 
> **데이터 타입이 ‘날짜(DATS)’, 시간(TIMS)’ 등인 경우 자동으로 내장된 도움말이 뜸.**
> 

**예시:**

| 필드 | 데이터 타입 | F4 동작 |
| --- | --- | --- |
| `FLDATE` | DATS | 달력 캘린더 팝업 |
| `PASSBIRTH` | DATS | 생년월일 캘린더 팝업 |
| `ORDER_DATE` | DATS | 날짜 선택 가능 |

즉, 도메인에서 형식이 날짜형이므로 SAP이 기본 F4 캘린더를 제공.

---

### 4 “Explicit search help interface to field”

**뜻:**

> 이 필드에는 개발자가 직접 연결한 Search Help 가 존재한다.
> 
> 
> (표준 Search Help가 아니라, 명시적으로 지정된 사용자 정의 Search Help.)
> 

**예시:**

| 필드 | 연결된 Search Help | 설명 |
| --- | --- | --- |
| `BOOKID` | SBOOK | 예약 ID 검색 도우미 (예약 내역에서 선택 가능) |

즉, F4를 누르면 단순 목록이 아니라 **커스텀 검색 화면(Search Help Dialog)** 이 뜹니다.

검색 조건, 다중 선택, 필터링 등이 가능.

---

### 5 “Input help implemented with search help”

**뜻:**

> 이건 데이터 요소 수준에서 Search Help 객체가 연결되어 있어서
> 
> 
> F4를 누르면 그 Search Help가 동작함.
> 

예:

데이터 요소 `S_CUSTOMER` → Search Help `SCUSTOM` 연결

→ `CUSTOMID` 필드에서 F4 누르면 고객 목록 검색창이 뜸.
